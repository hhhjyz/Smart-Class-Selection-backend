# 跨组接口契约（C 组 → D / F）

> 本文是智能选课子系统对**其他组**的稳定对外契约，供 D 组（论坛交流）、F 组（成绩管理）开发同学对接。
> 自包含：对接方无需阅读 C 组其他文档即可完成集成。
> **字段变更需先经 C 组架构 + 消费方组长在本文提 PR 评审，禁止单方面改动。**

## 概述

| 项 | 值 |
| --- | --- |
| 服务容器 | `course-selection:8003`（最终值落 `.env`） |
| 路由前缀 | `/api/course-selection/v1` |
| 鉴权 | 网关校验 JWT 后透传 `X-User-ID`、`X-User-Role`、`X-Request-ID` 三个 header，下游直接读 |
| 业务错误码段 | `30xx` |
| 时间格式 | ISO 8601，UTC（`2026-04-22T10:00:00Z`） |

统一响应壳（所有 REST 接口一致）：

```json
{ "code": 0, "message": "Success", "data": { ... }, "trace_id": "req-..." }
```

`code == 0` 表示成功；非 0 为业务错误码，含义见文末错误码表。`trace_id` 回显请求的 `X-Request-ID`，用于跨组排障。

## REST 接口

### 1. 查询学生选课列表

**消费方**：D 组（论坛按学生选课关联课程板块）。

| 项 | 值 |
| --- | --- |
| Method | `GET` |
| Path | `/api/course-selection/v1/students/{student_id}/enrollments` |
| 权限 | `admin` / `teacher` 全量可见；`student` 仅可访问 `student_id == X-User-ID` |

**路径参数**

| 名称 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `student_id` | string | 是 | 学号，如 `S-3210123` |

**Query 参数**

| 名称 | 类型 | 必填 | 默认 | 说明 |
| --- | --- | --- | --- | --- |
| `semester` | string | 是 | — | 学期，如 `2026-1` |
| `status` | string | 否 | `enrolled` | 选课状态过滤，见枚举 |

**成功响应**（HTTP 200）

```json
{
  "code": 0,
  "message": "Success",
  "data": {
    "list": [
      {
        "enrollment_id": "01HXXXX...",
        "offering_id": "B-CS101-2026-1-01",
        "course_code": "CS101",
        "course_name": "软件工程",
        "teacher_id": "T-9001",
        "teacher_name": "张老师",
        "status": "enrolled",
        "stage": "add_drop",
        "enrolled_at": "2026-04-22T10:00:00Z"
      }
    ],
    "total": 1
  },
  "trace_id": "req-abc123"
}
```

**字段说明**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `enrollment_id` | string | 选课记录主键（UUID v7） |
| `offering_id` | string | 开课实例 ID（沿用 B 组主键） |
| `course_code` | string | 课程代码 |
| `course_name` | string | 课程名 |
| `teacher_id` / `teacher_name` | string | 任课教师 |
| `status` | enum | `enrolled` / `pending_lottery` / `waitlisted` / `canceled` / `failed` |
| `stage` | enum | `preference` / `lottery` / `add_drop` |
| `enrolled_at` | string | 选课成功时间 |

**失败响应示例**（学生越权访问他人选课，HTTP 403）

```json
{ "code": 30003, "message": "角色越权", "data": null, "trace_id": "req-abc123" }
```

**调用示例**

```bash
curl -X GET "http://course-selection:8003/api/course-selection/v1/students/S-3210123/enrollments?semester=2026-1&status=enrolled" \
  -H "X-User-ID: S-3210123" \
  -H "X-User-Role: student" \
  -H "X-Request-ID: req-abc123"
```

### 2. 查询课程花名册

**消费方**：F 组（成绩录入获取花名册）。

| 项 | 值 |
| --- | --- |
| Method | `GET` |
| Path | `/api/course-selection/v1/offerings/{offering_id}/roster` |
| 权限 | 本课教师或 `admin` |

**路径参数**

| 名称 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `offering_id` | string | 是 | 开课实例 ID，如 `B-CS101-2026-1-01` |

**Query 参数**

| 名称 | 类型 | 必填 | 默认 | 说明 |
| --- | --- | --- | --- | --- |
| `include_dropped` | boolean | 否 | `false` | 是否含已退课学生 |

**成功响应**（HTTP 200）

```json
{
  "code": 0,
  "message": "Success",
  "data": {
    "offering_id": "B-CS101-2026-1-01",
    "course_code": "CS101",
    "semester": "2026-1",
    "students": [
      { "student_id": "S-3210123", "name": "李同学", "enrolled_at": "2026-04-22T10:00:00Z" }
    ],
    "total": 45,
    "snapshot_at": "2026-05-10T08:00:00Z"
  },
  "trace_id": "req-def456"
}
```

> 说明：`total` 为花名册学生数；`snapshot_at` 为本次查询读取时刻。当前接口**不分页**——单门课容量通常 < 300，全量返回。若后续出现超大班级需求，将以兼容方式新增 `page`/`page_size`，届时本文同步。

**失败响应示例**（非本课教师，HTTP 403）

```json
{ "code": 30003, "message": "角色越权", "data": null, "trace_id": "req-def456" }
```

**调用示例**

```bash
curl -X GET "http://course-selection:8003/api/course-selection/v1/offerings/B-CS101-2026-1-01/roster?include_dropped=false" \
  -H "X-User-ID: T-9001" \
  -H "X-User-Role: teacher" \
  -H "X-Request-ID: req-def456"
```

## 异步事件（RabbitMQ）

C 组通过 Outbox 模式向 D / F 组发布选课领域事件。这是跨组最终一致性的唯一通路。

### 连接信息

| 项 | 值 |
| --- | --- |
| Exchange | `enrollment.events` |
| Exchange 类型 | `topic` |
| 投递语义 | **at-least-once**（可能重复投递） |
| 幂等键 | 消费方按 `enrollment_id + occurred_at` 去重 |
| 连接串 | AMQP URI 由全系统共享 `.env`：`amqp://stss:***@rabbitmq:5672/` |

> 消费方自行声明 durable queue 并绑定到 `enrollment.events`，按需选择 routing key。重复投递必然发生（at-least-once），消费逻辑必须幂等。

### 事件：`enrollment.created`

| 项 | 值 |
| --- | --- |
| Routing key | `enrollment.created` |
| 触发条件 | 学生在任意阶段成功选上一门课 |
| 消费方 | D 组（更新关联板块）、F 组（预占花名册） |

**Payload schema**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `enrollment_id` | string | 选课记录主键 |
| `student_id` | string | 学号 |
| `offering_id` | string | 开课实例 ID |
| `semester` | string | 学期 |
| `stage` | enum | `preference` / `lottery` / `add_drop` |
| `occurred_at` | string | 事件发生时间（幂等键组成部分） |

**Payload 示例**

```json
{
  "enrollment_id": "01HXXXX...",
  "student_id": "S-3210123",
  "offering_id": "B-CS101-2026-1-01",
  "semester": "2026-1",
  "stage": "add_drop",
  "occurred_at": "2026-04-22T10:00:00Z"
}
```

### 事件：`enrollment.canceled`

| 项 | 值 |
| --- | --- |
| Routing key | `enrollment.canceled` |
| 触发条件 | 学生退课，或抽签未中签作废 |
| 消费方 | D 组、F 组 |

**Payload schema**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `enrollment_id` | string | 选课记录主键 |
| `student_id` | string | 学号 |
| `offering_id` | string | 开课实例 ID |
| `reason_code` | string | 退课原因码（`student_drop` / `lottery_lost` / `admin_revoke`） |
| `occurred_at` | string | 事件发生时间 |

**Payload 示例**

```json
{
  "enrollment_id": "01HXXXX...",
  "student_id": "S-3210123",
  "offering_id": "B-CS101-2026-1-01",
  "reason_code": "student_drop",
  "occurred_at": "2026-04-25T14:30:00Z"
}
```

### 事件：`lottery.completed`

| 项 | 值 |
| --- | --- |
| Routing key | `lottery.completed` |
| 触发条件 | 一次抽签批处理完成 |
| 消费方 | F 组（提示可拉取花名册） |

**Payload schema**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `run_id` | string | 抽签批次 ID |
| `semester` | string | 学期 |
| `succeeded_count` | int | 中签人数 |
| `failed_count` | int | 未中签人数 |
| `occurred_at` | string | 事件发生时间 |

**Payload 示例**

```json
{
  "run_id": "01HLOTTERY...",
  "semester": "2026-1",
  "succeeded_count": 1200,
  "failed_count": 340,
  "occurred_at": "2026-04-20T02:00:00Z"
}
```

### 消费端绑定示例（Python / aio-pika）

```python
import aio_pika

connection = await aio_pika.connect_robust("amqp://stss:***@rabbitmq:5672/")
channel = await connection.channel()
exchange = await channel.declare_exchange("enrollment.events", aio_pika.ExchangeType.TOPIC, durable=True)

# F 组：订阅创建 + 抽签完成
queue = await channel.declare_queue("grades.enrollment", durable=True)
await queue.bind(exchange, routing_key="enrollment.created")
await queue.bind(exchange, routing_key="enrollment.canceled")
await queue.bind(exchange, routing_key="lottery.completed")

async with queue.iterator() as it:
    async for message in it:
        async with message.process():
            payload = json.loads(message.body)
            # 幂等：按 (enrollment_id, occurred_at) 去重后再处理
            handle(payload)
```

## 跨组相关错误码

| Code | HTTP | 含义 |
| --- | --- | --- |
| 0 | 200 | 成功 |
| 30001 | 400 | 请求参数非法（如缺 `semester`） |
| 30002 | 401 | 网关身份头缺失 |
| 30003 | 403 | 角色越权 |
| 30004 | 404 | 资源不存在（学生 / 开课实例不存在） |

完整 30xx 错误码见《03 API 设计》§6 与《11 前端接口文档》。
